Work Out Training System

-- 2014 ARM Design Contest 2nd place--

Work Out Training Tracking System from Justin Wu on Vimeo.

Introduction

The work out training system can track user's motion using color detection. User needs to wear a specific color of wrist band, therefore, the system can track users' motion by webcam.
This project is implemented using C++ with OpenCV library. Moreover, we wrote some Verilog to create FPGA blocks trying to accelerate the calculation process. In order to let our software communicate with FPGA blocks, we developed the Linux kernel driver which utilize DMA on ARM Cortex A platform.
Date: July 2013

Music App Redisgn

Usability & Info. Architecture Final Project

The Goal

Nowadays, for music lovers, everyone uses apps listening to music. There are lots of different variety of music apps. Each one has its own design in the market. Therefore, our goal is to come up with a mobile application design that aims to improve the music management experience. Our design provides more friendly of using a music app. And it provides a beautiful and intuitive way for users to develop strong and long-lasting connections.

User Research

In order to re-design the layout, we had serveal interviews to investigate user's experience. Therefore, we came out of 10 different scenarios which include most of the general user process. Next we chose five specific scenario:

  • Creating a playlist to share music with people
  • Creating a playlist from another playlist
  • Create or Play a playlist before driving.
  • Create a playlist for top50 songs for this week.
  • Add a song from a specific artist into one of his/her playlist

Storyboard

Focusing on these 5 scenarios that are useful, we discussed about the key design. And then we draw out the storyboard to visualize specific use cases for our app.

Prototyping

Once we finished the storyboard, we were ready to prototype. We use Axure to design the page and gather them using invision. After that we did a comparative analyze with other different platform. Our design has a better page level and is friendlier than any other music apps.

  • User Scenario
  • Design Decisions
  • Comparative Analysis
